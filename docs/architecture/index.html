Notes App Architecture

src/ 디렉토리 기준의 컴포넌트 의존성, 상태 흐름, API 경계를 Mermaid로 정리한 문서입니다.

Entry

main.tsx가 React 루트를 만들고 App을 렌더링합니다.

State

App은 화면 상태, NotesProvider는 데이터 상태를 소유합니다.

API

api/notes.ts가 json-server /notes와 통신합니다.

src 구조

소스는 entry, app shell, context, api, components, types 레이어로 나뉩니다.

flowchart TD
  Src["src/"]
  Src --> Main["main.tsx\nReact root"]
  Src --> App["App.tsx\nscreen state + composition"]
  Src --> Context["context/NotesContext.tsx\nglobal notes state"]
  Src --> Api["api/notes.ts\nfetch CRUD"]
  Src --> Components["components/"]
  Src --> Types["types/note.ts\nNote shape"]
  Components --> Layout["Layout.tsx"]
  Components --> NoteList["NoteList.tsx"]
  Components --> NoteItem["NoteItem.tsx"]
  Components --> NoteEditor["NoteEditor.tsx"]

컴포넌트 의존성

Layout은 NoteList, NoteEditor를 직접 import하지 않고 sidebar, main 슬롯으로 받습니다. 

flowchart TD
  Main["main.tsx"] --> App["App.tsx"]
  App --> Provider["NotesProvider"]
  App --> Layout["Layout"]
  App --> NoteList["NoteList"]
  App --> NoteEditor["NoteEditor"]

  App -. "sidebar prop" .-> Layout
  App -. "main prop" .-> Layout
  Layout --> Header["Header + New Note button"]
  Layout --> SidebarSlot["sidebar slot"]
  Layout --> MainSlot["main slot"]

  NoteList --> UseNotesList["useNotes()"]
  NoteList --> NoteItem["NoteItem"]
  NoteEditor --> UseNotesEditor["useNotes()"]

  Provider --> NotesContext["NotesContext"]
  Provider --> NotesApi["api/notes.ts"]
  Provider --> NoteType["types/note.ts"]
  NotesApi --> JsonServer["json-server\nhttp://localhost:3001/notes"]
  NotesApi --> NoteType
  NoteEditor -. "reads Note shape via context" .-> NoteType
  NoteItem --> NoteType

상태 흐름

화면 선택 상태는 App에서 내려가고, 노트 데이터 상태는 NotesProvider에서 Context로 제공됩니다. 

flowchart LR
  AppState["App state\nselectedNoteId\nisCreating"]
  ProviderState["NotesProvider state\nnotes\nloading\nerror"]
  ContextValue["Context value\nnotes + create/update/delete"]
  Fetch["mount effect\nfetchNotes()"]

  AppState --> Layout["Layout"]
  AppState --> NoteList["NoteList"]
  AppState --> NoteEditor["NoteEditor"]

  Layout --> NewClick["+ 새 노트 클릭"]
  NewClick --> HandleNew["handleNewNote"]
  HandleNew --> AppState

  NoteItem["NoteItem 클릭"] --> HandleSelect["handleSelectNote"]
  HandleSelect --> AppState

  NoteEditor --> Done["onDone"]
  Done --> HandleDone["handleDone"]
  HandleDone --> AppState

  Fetch --> ProviderState
  ProviderState --> ContextValue
  ContextValue --> NoteList
  ContextValue --> NoteEditor

API / 데이터 변경 흐름

생성, 수정, 삭제는 Context 액션에서 API 계층을 호출한 뒤 성공 결과를 로컬 notes 배열에 반영합니다. 

flowchart TD
  NoteEditor["NoteEditor"] --> CreateAction["createNote(title, content)"]
  NoteEditor --> UpdateAction["updateNote(id, updates)"]
  NoteList["NoteList"] --> NoteItem["NoteItem"]
  NoteList -. "passes deleteNote prop" .-> NoteItem
  NoteItem --> DeleteAction["deleteNote(id)"]

  CreateAction --> ApiCreate["api.createNote\nPOST /notes"]
  UpdateAction --> ApiUpdate["api.updateNote\nPATCH /notes/:id"]
  DeleteAction --> ApiDelete["api.deleteNote\nDELETE /notes/:id"]

  ApiCreate --> Server["json-server /notes"]
  ApiUpdate --> Server
  ApiDelete --> Server

  Server --> SetNotes["setNotes"]
  SetNotes --> NotesProvider["NotesProvider"]
  NotesProvider --> UseNotes["useNotes consumers"]
