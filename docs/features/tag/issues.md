# 태그 기능 이슈

## 분해 원칙

- 각 이슈는 사용자가 노트 상세/편집 화면에서 확인할 수 있는 하나의 태그 동작을 완성한다.
- 각 이슈는 앞선 이슈의 결과를 기반으로 태그 표시, 편집, 검증, 저장 범위를 점진적으로 확장한다.
- 태그 별도 엔티티, 태그 검색/필터링, 노트 목록 태그 표시, 추천/자동완성, 색상 커스터마이징 등 PRD의 Out of Scope는 포함하지 않는다.
- AC는 이슈의 핵심 동작 검증에 필요한 범위로 제한한다.

## Issue 1. 저장된 태그 표시

### 설명

`Note` 데이터가 `tags` 값을 가질 수 있도록 하고, 노트 상세/편집 화면에서 저장된 태그를 제목 아래와 본문 위에 칩 형태로 표시한다. 기존 노트처럼 `tags` 필드가 없거나 빈 배열인 경우에도 편집 화면은 오류 없이 유지한다.

### 의존성

- 없음

### 완료조건

#### AC 1. 저장된 태그 칩 표시

- **Given** 선택된 노트에 `tags: ["업무", "긴급"]`이 저장되어 있고
- **When** 사용자가 해당 노트를 열면
- **Then** 제목 아래와 본문 위에 `업무`, `긴급` 태그 칩이 표시된다.

#### AC 2. 태그 없는 노트 표시

- **Given** 선택된 노트의 `tags`가 빈 배열이고
- **When** 사용자가 해당 노트를 열면
- **Then** 태그 칩은 표시되지 않고 제목, 본문, 저장 버튼이 정상 표시된다.

#### AC 3. 기존 노트 방어 처리

- **Given** 선택된 기존 노트에 `tags` 필드가 없고
- **When** 사용자가 해당 노트를 열면
- **Then** 태그가 없는 노트로 처리되어 화면 오류 없이 편집 화면이 표시된다.

## Issue 2. Enter로 태그 추가와 입력 검증

### 설명

태그 입력창을 추가하고 사용자가 Enter를 눌러 편집 중인 태그 목록에 새 태그를 추가할 수 있게 한다. 추가된 태그는 즉시 칩으로 표시되지만 저장 버튼을 누르기 전에는 서버에 반영하지 않는다. 태그 추가 시 빈 값, 중복, 길이 초과, 개수 초과를 검증하고 잘못된 입력은 추가하지 않는다.

### 의존성

- Issue 1. 저장된 태그 표시

### 완료조건

#### AC 1. 태그 입력창 표시

- **Given** 노트 상세/편집 화면이 열려 있고
- **When** 사용자가 제목 아래와 본문 위 영역을 확인하면
- **Then** 새 태그를 입력할 수 있는 accessible name이 있는 태그 입력창이 표시된다.

#### AC 2. Enter로 태그 추가

- **Given** 태그 입력창이 비어 있고
- **When** 사용자가 `React`를 입력한 뒤 Enter를 누르면
- **Then** 편집 중인 태그 목록에 `React` 태그 칩이 표시되고 입력창은 비워진다.

#### AC 3. 태그 정규화

- **Given** 태그 입력창이 비어 있고
- **When** 사용자가 `  React   Query  `를 입력한 뒤 Enter를 누르면
- **Then** 편집 중인 태그 목록에는 `React Query` 태그 칩이 표시된다.

#### AC 4. Enter가 노트 저장을 실행하지 않음

- **Given** 사용자가 노트 편집 중이고 태그 입력창에 `회의`를 입력했고
- **When** 사용자가 Enter를 누르면
- **Then** 태그만 편집 중인 목록에 추가되고 노트 저장 함수는 호출되지 않는다.

#### AC 5. 빈 태그 무시

- **Given** 태그 입력창이 비어 있거나 공백만 입력되어 있고
- **When** 사용자가 Enter를 누르면
- **Then** 새 태그 칩은 추가되지 않고 기존 태그 목록은 변경되지 않는다.

#### AC 6. 중복 태그 오류

- **Given** 편집 중인 태그 목록에 `React`가 있고
- **When** 사용자가 태그 입력창에 ` react `를 입력한 뒤 Enter를 누르면
- **Then** 새 태그는 추가되지 않고 입력창 아래에 중복 태그 inline 오류 메시지가 표시된다.

#### AC 7. 길이 초과 오류

- **Given** 태그 입력창이 비어 있고
- **When** 사용자가 16자 이상의 태그를 입력한 뒤 Enter를 누르면
- **Then** 새 태그는 추가되지 않고 입력창 아래에 15자 이하 입력 inline 오류 메시지가 표시된다.

#### AC 8. 개수 초과 오류

- **Given** 편집 중인 태그 목록에 이미 태그가 10개 있고
- **When** 사용자가 새 태그를 입력한 뒤 Enter를 누르면
- **Then** 새 태그는 추가되지 않고 입력창 아래에 최대 10개 inline 오류 메시지가 표시된다.

## Issue 3. 태그 삭제와 취소

### 설명

각 태그 칩에 삭제 버튼을 제공해 편집 중인 태그를 제거할 수 있게 한다. 취소 버튼을 누르면 저장하지 않은 태그 추가와 삭제를 모두 폐기하고 마지막 저장 상태로 되돌린다.

### 의존성

- Issue 2. Enter로 태그 추가와 입력 검증

### 완료조건

#### AC 1. 태그 칩 삭제

- **Given** 편집 중인 태그 목록에 `업무` 태그가 표시되어 있고
- **When** 사용자가 `업무` 태그의 삭제 버튼을 누르면
- **Then** 편집 중인 태그 목록에서 `업무` 태그 칩이 사라진다.

#### AC 2. 삭제 버튼 접근성

- **Given** 편집 중인 태그 목록에 `긴급` 태그가 표시되어 있고
- **When** 사용자가 보조 기술 또는 키보드로 태그 삭제 버튼을 탐색하면
- **Then** 삭제 대상 태그 이름을 포함한 accessible name으로 삭제 버튼을 확인할 수 있다.

#### AC 3. 취소 시 추가한 태그 폐기

- **Given** 저장된 태그 목록이 `["업무"]`이고 사용자가 `긴급` 태그를 추가했고
- **When** 사용자가 취소 버튼을 누른 뒤 같은 노트를 다시 확인하면
- **Then** `긴급` 태그는 표시되지 않고 저장된 `업무` 태그만 표시된다.

#### AC 4. 취소 시 삭제한 태그 복원

- **Given** 저장된 태그 목록이 `["업무", "긴급"]`이고 사용자가 `긴급` 태그를 삭제했고
- **When** 사용자가 취소 버튼을 누른 뒤 같은 노트를 다시 확인하면
- **Then** `업무`, `긴급` 태그가 다시 표시된다.

## Issue 4. 태그 변경 저장

### 설명

새 노트 생성과 기존 노트 수정 시 제목, 본문, 태그를 하나의 저장 동작으로 함께 서버에 반영한다. 저장 후 같은 노트를 다시 열면 마지막으로 저장한 태그 목록이 표시된다.

### 의존성

- Issue 3. 태그 삭제와 취소

### 완료조건

#### AC 1. 기존 노트 태그 저장

- **Given** 기존 노트의 저장된 태그 목록이 `["업무"]`이고 사용자가 `긴급` 태그를 추가했고
- **When** 사용자가 저장 버튼을 누르면
- **Then** 해당 노트 저장 요청에는 제목, 본문과 함께 `tags: ["업무", "긴급"]`가 포함된다.

#### AC 2. 기존 노트 태그 삭제 저장
  
- **Given** 기존 노트의 저장된 태그 목록이 `["업무", "긴급"]`이고 사용자가 `긴급` 태그를 삭제했고
- **When** 사용자가 저장 버튼을 누르면
- **Then** 해당 노트 저장 요청에는 제목, 본문과 함께 `tags: ["업무"]`가 포함된다.

#### AC 3. 새 노트 태그 저장

- **Given** 사용자가 새 노트를 작성하면서 제목, 본문, `React` 태그를 입력했고
- **When** 사용자가 저장 버튼을 누르면
- **Then** 새 노트 생성 요청에는 제목, 본문과 함께 `tags: ["React"]`가 포함된다.

#### AC 4. 저장 후 태그 표시 유지

- **Given** 사용자가 기존 노트에 `회의` 태그를 추가해 저장했고
- **When** 저장 완료 후 해당 노트를 다시 열면
- **Then** `회의` 태그 칩이 저장된 태그로 표시된다.

#### AC 5. 태그 입력 오류가 저장을 막지 않음

- **Given** 사용자가 유효한 제목, 본문, 태그를 편집했고 태그 입력창에는 마지막 오류 메시지가 표시되어 있고
- **When** 사용자가 저장 버튼을 누르면
- **Then** 현재 편집 중인 유효한 제목, 본문, 태그가 저장된다.
