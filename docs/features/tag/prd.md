# 태그 기능 PRD

## 1. 개요

### 1.1. 목적

사용자가 노트에 문자열 태그를 추가하고 삭제하여 노트의 주제나 성격을 간단히 표현할 수 있도록 한다.

태그는 노트 상세/편집 화면에서 제목과 본문 사이의 보조 정보로 제공한다. 태그 변경 사항은 제목 및 본문과 동일하게 노트 저장 시점에 함께 반영하며, 저장 전에는 편집 중인 상태로 유지한다.

### 1.2. 범위

- 노트 상세/편집 화면에서 현재 태그를 칩 형태로 표시한다.
- 태그 입력창에서 Enter를 눌러 태그를 추가한다.
- 태그 칩의 삭제 버튼으로 태그를 제거한다.
- 저장 버튼을 누르면 제목, 본문, 태그 변경 사항을 함께 저장한다.
- 취소 버튼을 누르면 저장하지 않은 태그 변경 사항을 폐기한다.
- 태그는 선택 입력값이며 태그가 없는 노트도 저장할 수 있다.
- 입력한 태그는 앞뒤 공백을 제거하고 내부의 연속 공백을 한 칸으로 축약한다.
- 사용자가 입력한 태그의 대소문자는 표시 및 저장 시 보존한다.
- 중복 여부는 정규화된 태그를 기준으로 대소문자를 구분하지 않고 판단한다.
- 빈 태그는 추가하지 않는다.
- 태그 하나의 길이는 최대 15자로 제한한다.
- 노트 하나의 태그 개수는 최대 10개로 제한한다.
- 잘못된 태그 입력은 추가하지 않고 입력창 아래에 짧은 inline 오류 메시지를 표시한다.
- 태그 입력 오류가 있더라도 이미 유효한 노트 내용과 태그의 저장은 막지 않는다.
- 기존 노트에 태그 정보가 없더라도 태그가 없는 노트로 정상 표시한다.

## 2. 사용자 스토리

### 2.1. 태그 확인

**노트를 작성하거나 편집하는 사용자로서**, 현재 노트에 지정된 태그를 확인하고 싶다. **그래야** 노트가 어떤 주제나 성격으로 분류되어 있는지 편집 화면에서 바로 이해할 수 있다.

- 태그는 노트 상세/편집 화면의 제목 아래, 본문 위에 표시된다.
- 태그는 서로 구분하기 쉬운 칩 형태로 표시된다.
- 태그가 없는 경우 별도의 큰 빈 상태 화면을 표시하지 않는다.

### 2.2. 태그 추가

**노트를 작성하거나 편집하는 사용자로서**, 입력창에 태그를 입력하고 Enter로 추가하고 싶다. **그래야** 노트의 주제나 성격을 빠르게 기록할 수 있다.

- Enter 입력은 태그 추가에만 사용되며 노트 저장을 실행하지 않는다.
- 태그의 앞뒤 공백과 연속된 내부 공백은 추가 전에 정규화된다.
- 정규화 결과가 빈 문자열이면 태그를 추가하지 않는다.
- 유효한 태그는 편집 중인 태그 목록에 즉시 표시되지만 서버에는 즉시 저장되지 않는다.

### 2.3. 태그 삭제

**노트를 작성하거나 편집하는 사용자로서**, 더 이상 필요하지 않은 태그를 제거하고 싶다. **그래야** 현재 노트에 맞는 태그만 유지할 수 있다.

- 각 태그 칩은 삭제 기능을 제공한다.
- 삭제 버튼은 태그 칩에 마우스를 올렸을 때 표시된다.
- 삭제 결과는 편집 중인 상태에 먼저 반영되며 서버에는 즉시 저장되지 않는다.

### 2.4. 태그 변경 저장

**노트를 작성하거나 편집하는 사용자로서**, 제목과 본문을 저장할 때 태그 변경 사항도 함께 저장하고 싶다. **그래야** 노트의 모든 편집 내용을 하나의 명확한 저장 동작으로 반영할 수 있다.

- 저장 버튼을 누르면 제목, 본문, 태그를 함께 저장한다.
- 태그를 추가하거나 삭제한 뒤 저장하지 않으면 서버 데이터는 변경되지 않는다.
- 태그가 없어도 노트를 정상적으로 저장할 수 있다.

### 2.5. 태그 변경 취소

**노트를 작성하거나 편집하는 사용자로서**, 저장하지 않은 태그 변경을 취소하고 싶다. **그래야** 실수로 추가하거나 삭제한 태그를 기존 저장 상태로 되돌릴 수 있다.

- 취소 버튼을 누르면 저장 이후에 이루어진 태그 추가 및 삭제를 모두 폐기한다.
- 취소 후에는 마지막으로 저장된 태그 목록을 표시한다.

### 2.6. 유효하지 않은 태그 입력 확인

**노트를 작성하거나 편집하는 사용자로서**, 태그를 추가할 수 없는 이유를 바로 확인하고 싶다. **그래야** 입력을 수정해 유효한 태그를 추가할 수 있다.

- 기존 태그와 대소문자만 다른 태그는 중복으로 판단해 추가하지 않는다.
- 정규화된 태그가 15자를 초과하면 추가하지 않는다.
- 이미 태그가 10개인 경우 새 태그를 추가하지 않는다.
- 오류 사유는 태그 입력창 아래에 짧은 inline 메시지로 표시한다.
- 오류가 발생해도 기존의 유효한 태그 목록은 변경하지 않는다.
- 오류가 발생해도 이미 유효한 노트 내용과 태그를 저장할 수 있다.

### 2.7. 기존 노트 열기

**기존 노트를 사용하는 사용자로서**, 태그 기능 도입 전에 작성한 노트도 오류 없이 열고 편집하고 싶다. **그래야** 기존 데이터를 그대로 유지하면서 새 태그 기능을 사용할 수 있다.

- 태그 정보가 없는 기존 노트는 태그가 없는 상태로 표시한다.
- 기존 노트에도 새 태그를 추가해 저장할 수 있다.

## 4. Out of Scope

### 4.1. 태그 데이터의 독립 관리

- 태그를 노트와 분리된 독립 엔티티로 생성하거나 관리하는 기능
- 태그마다 별도의 식별자, 설명, 색상, 아이콘, 생성일, 수정일 등의 메타데이터를 부여하는 기능
- 전체 태그를 한 화면에서 조회, 생성, 수정, 삭제하는 별도 태그 관리 화면
- 태그 삭제 시 해당 태그가 사용된 모든 노트에서 일괄 제거하는 기능
- 태그 이름 변경 시 해당 태그가 사용된 모든 노트에 일괄 반영하는 기능
- 태그 간 상위/하위 관계나 계층 구조를 정의하는 기능
- 서로 다른 태그를 병합하거나 별칭으로 연결하는 기능
- 사용자 또는 워크스페이스 단위의 공용 태그 사전을 운영하는 기능

### 4.2. 노트 탐색 및 분류

- 태그 이름으로 노트를 검색하는 기능
- 특정 태그를 선택해 노트 목록을 필터링하는 기능
- 여러 태그를 AND 또는 OR 조건으로 조합해 노트를 필터링하는 기능
- 태그별 노트 목록이나 태그 전용 상세 페이지를 제공하는 기능
- 태그를 기준으로 노트를 정렬하거나 그룹화하는 기능
- URL 쿼리 또는 별도 라우트로 태그 필터 상태를 공유하거나 유지하는 기능
- 태그 기반의 폴더, 컬렉션, 스마트 목록을 생성하는 기능

### 4.3. 노트 목록 및 기타 화면 표시

- 노트 목록 항목에 태그를 표시하는 기능
- 노트 카드, 미리보기, 검색 결과 등 상세/편집 화면 이외의 위치에 태그를 표시하는 기능
- 사이드바나 내비게이션에 전체 태그 목록 또는 태그별 노트 개수를 표시하는 기능
- 태그가 없는 노트를 위한 별도의 빈 상태, 경고, 배지 또는 안내 화면
- 태그 추가 및 삭제만을 위한 별도 모달, 드로어 또는 전용 화면

### 4.4. 추천 및 자동화

- 최근 사용한 태그를 추천하는 기능
- 자주 사용한 태그나 인기 태그를 추천하는 기능
- 제목 또는 본문 내용을 분석해 태그를 자동 생성하거나 제안하는 기능
- 입력 중 자동완성 후보를 제공하는 기능
- 기존 태그 사전을 기반으로 유사 태그나 오타 교정을 제안하는 기능
- 노트 생성 시 기본 태그를 자동으로 부여하는 기능
- 규칙, 템플릿 또는 외부 서비스에 따라 태그를 자동 추가하거나 제거하는 기능

### 4.5. 고급 입력 및 편집 방식

- 쉼표, 탭, 공백, 붙여넣기 등의 입력으로 여러 태그를 한 번에 추가하는 기능
- 여러 노트를 선택해 태그를 일괄 추가하거나 삭제하는 기능
- 드래그 앤 드롭으로 태그 순서를 변경하는 기능
- 태그 칩을 직접 편집해 이름을 변경하는 기능
- 태그별 색상을 사용자가 지정하거나 변경하는 기능
- 모바일 전용 태그 입력 패턴이나 별도의 터치 상호작용을 추가하는 기능
- 키보드 방향키로 태그 후보를 탐색하는 등 자동완성 기반의 고급 키보드 조작

### 4.6. 저장 및 동기화 방식

- 태그를 추가하거나 삭제하는 즉시 서버에 자동 저장하는 기능
- 제목 및 본문과 분리해 태그만 별도로 저장하는 기능
- 저장하지 않은 태그 변경 사항을 다른 노트나 세션으로 이동한 뒤에도 임시 보존하는 기능
- 여러 사용자 또는 여러 기기에서 동시에 편집한 태그의 충돌을 병합하는 기능
- 태그 변경 이력, 감사 로그, 이전 버전 복원 기능
- 오프라인 상태에서 편집한 태그를 이후 자동 동기화하는 기능

### 4.7. 권한 및 협업

- 태그별 조회 또는 편집 권한을 설정하는 기능
- 특정 사용자나 팀만 사용할 수 있는 비공개 태그 기능
- 태그 변경에 대한 승인 절차나 관리자 검토 기능
- 태그를 이용한 사용자 멘션, 공유, 알림 기능
- 다른 사용자가 변경한 태그를 실시간으로 반영하는 협업 기능

### 4.8. 통계 및 외부 연동

- 태그별 노트 수, 사용 빈도, 최근 사용일 등의 통계 제공
- 태그 사용 현황을 차트나 대시보드로 시각화하는 기능
- 태그 데이터를 CSV, JSON 등으로 별도 내보내거나 가져오는 기능
- 외부 서비스의 태그 또는 라벨과 동기화하는 기능
- 공개 API, 웹훅 또는 플러그인에서 태그를 별도 관리하는 기능

## 5. 용어 정의

| 용어               | 정의                                                                                                                                        |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------- |
| 태그               | 노트의 주제나 성격을 표현하기 위해 노트에 추가하는 문자열 값이다.                                                                           |
| 태그 목록          | 하나의 노트에 현재 지정되어 있는 태그의 집합이다. 노트당 최대 10개까지 허용한다.                                                            |
| 태그 입력창        | 사용자가 새 태그 문자열을 입력하고 Enter로 추가하는 입력 영역이다.                                                                          |
| 태그 칩            | 추가된 태그를 화면에 구분하여 표시하고 삭제 동작을 제공하는 UI 요소다.                                                                      |
| 정규화             | 태그를 추가하기 전에 앞뒤 공백을 제거하고 내부의 연속 공백을 한 칸으로 축약하는 처리다.                                                     |
| 중복 태그          | 정규화한 문자열을 대소문자 구분 없이 비교했을 때 이미 같은 노트에 존재하는 태그다. 예를 들어 `React`, `react`, `REACT`는 중복으로 판단한다. |
| 편집 상태          | 사용자가 태그를 추가하거나 삭제했지만 아직 저장하지 않아 서버 데이터에는 반영되지 않은 화면 내 상태다.                                      |
| 저장               | 제목, 본문, 태그의 현재 편집 상태를 하나의 노트 데이터로 서버에 반영하는 동작이다.                                                          |
| 취소               | 마지막 저장 이후의 제목, 본문, 태그 변경 사항을 폐기하고 저장된 상태로 되돌리는 동작이다.                                                   |
| inline 오류 메시지 | 태그 입력 오류의 원인을 입력창 가까이에 짧은 문장으로 표시하는 피드백이다.                                                                  |
| 기존 노트          | 태그 기능이 도입되기 전에 생성되어 태그 정보가 없을 수 있는 노트다. 태그가 없는 상태로 처리한다.                                            |
