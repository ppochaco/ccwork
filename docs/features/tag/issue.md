# 태그 기능 이슈

## 분해 원칙

- 수직 슬라이싱 원칙에 따라 각 이슈는 특정 기술 계층이 아니라 사용자가 확인할 수 있는 하나의 동작을 완성한다.
- 앞선 이슈가 완료된 상태를 기반으로 다음 사용자 가치를 점진적으로 추가한다.
- 태그 검색, 필터, 추천, 독립 관리 등 PRD의 Out of Scope는 포함하지 않는다.

## Issue 1. 태그 조회

### 설명

노트 상세/편집 화면의 제목과 본문 사이에 저장된 태그를 칩 형태로 표시한다. 애플리케이션 내부에서는 모든 노트가 `tags` 배열을 갖도록 데이터 모델과 API 응답 경계를 정리한다. 태그 기능 도입 전에 생성되어 `tags` 필드가 없는 노트도 빈 태그 목록으로 처리하여 기존 데이터와의 호환성을 보장한다.

### 완료조건

#### AC 1. 저장된 태그 표시

- **Given** 태그가 저장된 기존 노트가 있고
- **When** 사용자가 해당 노트를 편집 화면에서 열면
- **Then** 저장된 태그가 제목 아래, 본문 위에 각각 구분 가능한 칩으로 표시된다.

#### AC 2. 태그가 없는 노트 표시

- **Given** `tags`가 빈 배열인 노트가 있고
- **When** 사용자가 해당 노트를 편집 화면에서 열면
- **Then** 태그 칩은 표시되지 않으며 별도의 큰 빈 상태 화면 없이 노트를 정상적으로 편집할 수 있다.

#### AC 3. 기존 데이터 하위 호환

- **Given** API 응답에 `tags` 필드가 없는 기존 노트가 있고
- **When** 애플리케이션이 노트를 조회하면
- **Then** 해당 노트는 `tags: []`인 노트로 변환되어 오류 없이 표시된다.

#### AC 4. 표시 위치 제한

- **Given** 태그가 저장된 노트가 있고
- **When** 사용자가 노트 목록과 노트 편집 화면을 확인하면
- **Then** 태그는 노트 편집 화면에만 표시되고 노트 목록에는 표시되지 않는다.

## Issue 2. 태그 추가

### 설명

사용자가 태그 입력창에 값을 입력하고 Enter를 눌러 편집 중인 태그 목록에 추가할 수 있게 한다. 추가 전 공백을 정규화하되 사용자가 입력한 대소문자는 보존한다. 추가된 태그는 편집 상태에 먼저 반영하고, 저장 버튼을 누르면 제목 및 본문과 함께 서버에 반영한다.

### 완료조건

#### AC 1. Enter로 태그 추가

- **Given** 사용자가 노트 작성 또는 편집 화면의 태그 입력창에 유효한 태그를 입력했고
- **When** Enter를 누르면
- **Then** 태그가 편집 중인 태그 목록에 칩으로 추가되고 노트 저장은 실행되지 않는다.

#### AC 2. 태그 공백 정규화

- **Given** 사용자가 앞뒤 공백과 연속된 내부 공백이 포함된 `  프로젝트   계획  `을 입력했고
- **When** Enter를 누르면
- **Then** 태그는 `프로젝트 계획`으로 정규화되어 추가되며 사용자가 입력한 문자의 대소문자는 보존된다.

#### AC 3. 빈 태그 무시

- **Given** 태그 입력값이 비어 있거나 공백만 포함하고 있고
- **When** 사용자가 Enter를 누르면
- **Then** 태그는 추가되지 않고 오류 메시지도 표시되지 않는다.

#### AC 4. 저장 전 서버 미반영

- **Given** 사용자가 유효한 태그를 편집 중인 목록에 추가했고
- **When** 저장 버튼을 누르기 전이면
- **Then** 태그 추가를 위한 API 요청은 발생하지 않는다.

#### AC 5. 추가한 태그 저장

- **Given** 사용자가 새 노트 또는 기존 노트에 유효한 태그를 추가했고
- **When** 저장 버튼을 누른 뒤 해당 노트를 다시 조회하면
- **Then** 제목, 본문, 전체 태그 목록이 하나의 노트 데이터로 저장되고 추가한 태그가 표시된다.

## Issue 3. 태그 삭제

### 설명

각 태그 칩에 해당 태그를 제거하는 삭제 버튼을 제공한다. 삭제는 편집 상태에 먼저 반영하고, 저장 버튼을 누르면 제목, 본문, 남은 태그 목록과 함께 서버에 반영한다. 삭제 버튼은 마우스와 키보드로 사용할 수 있고 삭제할 태그를 식별할 수 있는 이름을 제공한다.

### 완료조건

#### AC 1. 삭제 버튼 접근

- **Given** 편집 화면에 태그 칩이 표시되어 있고
- **When** 사용자가 칩에 마우스를 올리거나 삭제 버튼에 키보드 포커스를 이동하면
- **Then** 삭제 버튼을 확인하고 실행할 수 있으며 버튼 이름으로 삭제할 태그를 식별할 수 있다.

#### AC 2. 편집 상태에서 태그 삭제

- **Given** 노트에 여러 태그가 표시되어 있고
- **When** 사용자가 특정 태그의 삭제 버튼을 누르면
- **Then** 선택한 태그만 편집 중인 목록에서 제거되고 다른 태그는 유지된다.

#### AC 3. 저장 전 서버 미반영

- **Given** 사용자가 태그를 편집 중인 목록에서 삭제했고
- **When** 저장 버튼을 누르기 전이면
- **Then** 태그 삭제를 위한 API 요청은 발생하지 않고 서버의 태그 목록은 변경되지 않는다.

#### AC 4. 삭제한 태그 저장

- **Given** 사용자가 기존 노트의 태그를 하나 이상 삭제했고
- **When** 저장 버튼을 누른 뒤 해당 노트를 다시 조회하면
- **Then** 제목, 본문, 남은 태그 목록이 하나의 노트 데이터로 저장되고 삭제한 태그는 표시되지 않는다.

#### AC 5. 모든 태그 삭제 후 저장

- **Given** 사용자가 노트의 모든 태그를 삭제했고
- **When** 저장 버튼을 누르면
- **Then** 노트는 빈 태그 목록으로 정상 저장된다.

## Issue 4. 태그 변경 취소

### 설명

태그 추가와 삭제를 제목 및 본문과 동일한 임시 편집 상태로 관리한다. 사용자가 취소하면 저장하지 않은 변경을 서버에 반영하지 않고 마지막으로 저장된 태그 목록으로 돌아간다. 다른 노트 또는 새 노트 작성 화면으로 전환할 때도 대상에 맞는 태그 상태를 표시한다.

### 완료조건

#### AC 1. 태그 변경 취소

- **Given** 사용자가 마지막 저장 이후 태그를 추가하거나 삭제했고
- **When** 취소 버튼을 누른 뒤 같은 노트를 다시 열면
- **Then** 저장하지 않은 태그 변경은 폐기되고 마지막으로 저장된 태그 목록이 표시된다.

#### AC 2. 취소 시 서버 미반영

- **Given** 저장하지 않은 태그 변경이 있고
- **When** 사용자가 취소 버튼을 누르면
- **Then** 노트 생성 또는 수정 API 요청 없이 편집을 종료한다.

#### AC 3. 편집 대상 전환

- **Given** 사용자가 한 노트의 태그를 저장하지 않은 채 다른 노트 또는 새 노트 작성 화면으로 전환했고
- **When** 새 편집 대상이 표시되면
- **Then** 이전 임시 태그가 섞이지 않고 선택한 노트의 저장된 태그 또는 빈 태그 목록이 표시된다.

## Issue 5. 태그 입력 검증

### 설명

태그 추가 시 정규화된 값을 기준으로 중복, 길이, 개수 제한을 검증한다. 유효하지 않은 입력은 태그 목록을 변경하지 않고 입력창 가까이에 오류 사유를 표시한다. 태그 입력 오류는 태그 추가만 제한하며, 이미 유효한 제목, 본문, 태그의 저장을 막지 않는다.

### 완료조건

#### AC 1. 대소문자를 구분하지 않는 중복 검사

- **Given** 노트에 `React` 태그가 이미 있고
- **When** 사용자가 공백 정규화 후 같은 값인 `react` 또는 `REACT`를 추가하면
- **Then** 새 태그는 추가되지 않고 중복 태그 오류 메시지가 입력창 아래에 표시된다.

#### AC 2. 태그 길이 제한

- **Given** 사용자가 정규화 결과 15자를 초과하는 태그를 입력했고
- **When** Enter를 누르면
- **Then** 태그는 추가되지 않고 15자 이하로 입력해야 한다는 오류 메시지가 입력창 아래에 표시된다.

#### AC 3. 태그 개수 제한

- **Given** 편집 중인 노트에 태그가 이미 10개 있고
- **When** 사용자가 새로운 유효한 태그를 추가하면
- **Then** 태그는 추가되지 않고 최대 10개까지 추가할 수 있다는 오류 메시지가 입력창 아래에 표시된다.

#### AC 4. 오류 상태 표시

- **Given** 편집 중인 노트에 유효한 태그 목록이 있고
- **When** 사용자가 유효하지 않은 태그 추가를 시도하면
- **Then** 기존 태그 목록은 변경되지 않고 태그 입력은 `aria-invalid` 상태를 가지며 입력과 연결된 오류 메시지가 표시된다.

#### AC 5. 오류 복구와 노트 저장

- **Given** 태그 입력 오류가 표시되어 있고
- **When** 사용자가 유효한 태그를 추가하거나 노트 저장을 실행하면
- **Then** 유효한 태그 추가 시 입력값과 오류가 초기화되며, 노트 저장 시 유효하지 않은 입력값을 제외한 제목, 본문, 태그는 정상 저장된다.
