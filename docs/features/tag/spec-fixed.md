# 태그 기능 확정 정의서

## 기능 개요

노트에 문자열 태그를 추가하고 삭제할 수 있다.
태그는 노트 상세/편집 화면에서 확인하며, 노트 저장 시 제목/본문과 함께 저장한다.

## 선택 요약

- 태그는 `string[]`로 저장한다.
  - 이유: 원본 정의서가 문자열 태그와 중복 금지만 요구하므로 별도 엔티티 없이 노트 내부 배열로 관리하는 방식이 가장 단순하다.
- 태그는 별도 엔티티로 관리하지 않는다.
  - 이유: 전체 태그 목록, 태그 이름 일괄 변경, 태그별 통계가 현재 범위에 없으므로 데이터 모델을 크게 만들 필요가 없다.
- 태그 UI는 `NoteEditor` 제목 아래에 둔다.
  - 이유: 원본 정의서의 “노트 상세 화면에서 확인” 요구에 맞고, 현재 앱에서 상세/편집 화면 역할을 `NoteEditor`가 담당한다.
- 태그 추가/삭제는 저장 버튼을 눌렀을 때 함께 저장한다.
  - 이유: 제목과 본문도 저장 버튼으로 반영되는 구조라 편집 모델이 일관된다.
- 태그 입력은 Enter 추가와 칩 삭제 패턴을 사용한다.
  - 이유: 현재 폼 구조에 자연스럽게 붙고, 중복/길이/개수 검사를 명확히 처리할 수 있다.

## 대상 화면

- 태그 UI는 `NoteEditor`에 배치한다.
- 태그 영역은 제목 입력 아래, 본문 입력 위에 배치한다.
- 노트 목록(`NoteList`, `NoteItem`)에는 태그를 표시하지 않는다.
- 별도 태그 관리 화면이나 라우트를 만들지 않는다.

## 상태 및 데이터

`Note` 타입에 `tags: string[]` 필드를 추가한다.

```ts
interface Note {
  id: string;
  title: string;
  content: string;
  tags: string[];
  createdAt: string;
  updatedAt: string;
}
```

- 새 노트의 기본 태그 값은 빈 배열(`[]`)이다.
- 태그는 각 노트의 `tags` 배열에 문자열로 저장한다.
- 태그 편집 상태는 `NoteEditor` 내부에서 제목/본문과 함께 관리한다.
- 기존 노트에 `tags` 필드가 없으면 빈 배열로 처리한다.

## 주요 동작

- 사용자는 태그 입력창에 값을 입력하고 Enter로 태그를 추가한다.
- 태그 입력창에서 Enter를 눌러도 노트 저장은 실행하지 않는다.
- 사용자는 태그 칩의 삭제 버튼으로 태그를 제거한다.
- 태그 추가/삭제는 즉시 서버에 저장하지 않는다.
- 저장 버튼을 누르면 제목, 본문, 태그를 함께 저장한다.
- 취소 버튼을 누르면 저장하지 않은 태그 변경도 취소된다.
- 태그 없이도 노트를 저장할 수 있다.

## 입력 및 표시 규칙

- 태그는 칩 형태로 표시한다.
- 태그 칩에는 삭제 버튼을 둔다.
- 삭제 버튼은 태그 칩에 hover했을 때 보이도록 한다.
- 태그가 없으면 빈 상태를 크게 강조하지 않는다.
- 태그 영역은 제목과 본문 사이의 보조 정보로 취급한다.

태그를 추가할 때 다음 정규화를 적용한다.

- 앞뒤 공백을 제거한다.
- 내부의 연속 공백은 한 칸으로 축약한다.
- 사용자가 입력한 대소문자는 보존한다.

예시:

```text
"  React   Query  " -> "React Query"
```

## 검증 규칙

- 중복 태그는 허용하지 않는다.
- 중복 판단은 대소문자를 구분하지 않는다.
- 정규화된 문자열 기준으로 비교한다.
- 빈 태그는 추가하지 않고 무시한다.
- 태그 하나의 최대 길이는 15자이다.
- 노트 하나에 추가할 수 있는 태그는 최대 10개이다.
- 태그는 필수 입력값이 아니다.

중복 판단 예시:

```text
"React", "react", " REACT " -> 같은 태그
```

## 오류 및 예외 처리

- 태그 입력 오류는 태그 입력창 아래에 짧은 inline 메시지로 표시한다.
- 중복 태그를 추가하려는 경우 오류 메시지를 표시한다.
- 태그가 15자를 초과하는 경우 오류 메시지를 표시한다.
- 태그 개수가 10개를 초과하는 경우 오류 메시지를 표시한다.
- 오류가 있는 태그 입력은 추가하지 않는다.
- 이미 유효한 노트 내용과 태그 저장은 막지 않는다.
- `db.json`의 기존 노트에는 `tags: []`를 명시적으로 추가한다.
- 앱 코드에서는 `tags` 필드가 누락된 노트도 `[]`로 처리한다.

## 구현 범위

이번 기능에 포함한다.

- `Note` 타입에 `tags` 필드 추가
- 생성/수정 API 요청에 `tags` 포함
- `NoteEditor`에서 태그 입력, 표시, 삭제 처리
- 태그 정규화, 중복 검사, 길이 제한, 개수 제한
- 기존 `db.json` 데이터에 `tags: []` 추가
- 누락된 `tags` 필드에 대한 방어 처리

이번 기능에 포함하지 않는다.

- 전체 태그 목록 관리
- 태그 별도 엔티티
- 태그 검색/필터링
- 태그 이름 일괄 변경
- 노트 목록에서 태그 표시
- 최근 사용 태그 추천

## 수용 기준

- 사용자는 노트 상세/편집 화면에서 태그 목록을 확인할 수 있다.
- 사용자는 태그 입력창에 값을 입력하고 Enter로 태그를 추가할 수 있다.
- 사용자는 태그 칩의 삭제 버튼으로 태그를 제거할 수 있다.
- 저장 버튼을 누르면 태그 변경이 서버 데이터에 반영된다.
- 취소 버튼을 누르면 저장하지 않은 태그 변경이 반영되지 않는다.
- 같은 의미의 태그는 대소문자 차이가 있어도 중복으로 처리된다.
- 빈 태그, 15자 초과 태그, 10개 초과 태그는 추가되지 않는다.
- 태그 입력 오류는 inline 메시지로 확인할 수 있다.
- 기존 `tags` 필드가 없는 노트도 화면 오류 없이 표시된다.
