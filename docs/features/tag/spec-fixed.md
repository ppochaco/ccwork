# 태그 기능 확정 정의서

## 기능 개요

노트에 문자열 태그를 추가하고 삭제할 수 있다.
태그는 노트 상세/편집 화면에서 확인하며, 노트 저장 시 제목/본문과 함께 저장한다.

## 대상 화면

- `NoteEditor`에서 태그를 관리한다.
- 태그 영역은 제목 입력 아래, 본문 입력 위에 배치한다.
- 노트 목록(`NoteList`, `NoteItem`)에는 태그를 표시하지 않는다.

## 데이터 구조

`Note` 타입에 `tags: string[]` 필드를 추가한다.

```ts
interface Note {
  id: string;
  title: string;
  content: string;
  tags: string[];
  createdAt: string;
  updatedAt: string;
}
```

## 저장 정책

- 태그는 별도 엔티티로 관리하지 않는다.
- 태그는 각 노트의 `tags` 배열에 문자열로 저장한다.
- 새 노트의 기본 태그 값은 빈 배열(`[]`)이다.
- 태그 없이도 노트를 저장할 수 있다.
- 태그 추가/삭제는 즉시 서버에 저장하지 않는다.
- 저장 버튼을 누르면 제목, 본문, 태그를 함께 저장한다.
- 취소 버튼을 누르면 저장하지 않은 태그 변경도 취소된다.

## 입력 UI

- 태그 입력은 `입력창 + Enter 추가 + 태그 칩 삭제` 패턴을 사용한다.
- 태그 입력창에서 Enter를 누르면 태그만 추가한다.
- 태그 입력창 Enter는 노트 저장을 실행하지 않는다.
- 태그 칩에는 삭제 버튼을 둔다.
- 삭제 버튼은 태그 칩에 hover했을 때 보이도록 한다.

## 태그 표시

- 태그는 칩 형태로 표시한다.
- 태그가 없으면 빈 상태를 크게 강조하지 않는다.
- 태그 영역은 제목과 본문 사이의 보조 정보로 취급한다.

## 정규화 규칙

태그를 추가할 때 다음 정규화를 적용한다.

- 앞뒤 공백을 제거한다.
- 내부의 연속 공백은 한 칸으로 축약한다.
- 사용자가 입력한 대소문자는 보존한다.

예시:

```text
"  React   Query  " -> "React Query"
```

## 중복 처리

- 중복 태그는 허용하지 않는다.
- 중복 판단은 대소문자를 구분하지 않는다.
- 정규화된 문자열 기준으로 비교한다.

예시:

```text
"React", "react", " REACT " -> 같은 태그
```

## 제한 사항

- 빈 태그는 추가하지 않고 무시한다.
- 태그 하나의 최대 길이는 15자이다.
- 노트 하나에 추가할 수 있는 태그는 최대 10개이다.
- 태그는 필수 입력값이 아니다.

## 오류 처리

태그 입력 오류는 태그 입력창 아래에 짧은 inline 메시지로 표시한다.

오류 메시지를 표시하는 경우:

- 중복 태그를 추가하려는 경우
- 태그가 15자를 초과하는 경우
- 태그 개수가 10개를 초과하는 경우

오류가 있는 태그 입력은 추가하지 않는다.
다만 이미 유효한 노트 내용과 태그 저장은 막지 않는다.

## 기존 데이터 마이그레이션

- `db.json`의 기존 노트에는 `tags: []`를 명시적으로 추가한다.
- 앱 코드에서는 `tags` 필드가 누락된 노트도 `[]`로 처리한다.
- 이를 통해 샘플 데이터와 런타임 모두 새 스키마에 안전하게 대응한다.

## 구현 범위

이번 기능에 포함한다.

- `Note` 타입에 `tags` 필드 추가
- 생성/수정 API 요청에 `tags` 포함
- `NoteEditor`에서 태그 입력, 표시, 삭제 처리
- 태그 정규화, 중복 검사, 길이 제한, 개수 제한
- 기존 `db.json` 데이터에 `tags: []` 추가
- 누락된 `tags` 필드에 대한 방어 처리

이번 기능에 포함하지 않는다.

- 전체 태그 목록 관리
- 태그 별도 엔티티
- 태그 검색/필터링
- 태그 이름 일괄 변경
- 노트 목록에서 태그 표시
- 최근 사용 태그 추천

## 수용 기준

- 사용자는 노트 상세/편집 화면에서 태그 목록을 확인할 수 있다.
- 사용자는 태그 입력창에 값을 입력하고 Enter로 태그를 추가할 수 있다.
- 사용자는 태그 칩의 삭제 버튼으로 태그를 제거할 수 있다.
- 저장 버튼을 누르면 태그 변경이 서버 데이터에 반영된다.
- 취소 버튼을 누르면 저장하지 않은 태그 변경이 반영되지 않는다.
- 같은 의미의 태그는 대소문자 차이가 있어도 중복으로 처리된다.
- 빈 태그, 15자 초과 태그, 10개 초과 태그는 추가되지 않는다.
- 태그 입력 오류는 inline 메시지로 확인할 수 있다.
- 기존 `tags` 필드가 없는 노트도 화면 오류 없이 표시된다.
