# 검색 기능 확정 정의서

## 기능 개요

사용자는 검색어로 노트를 찾을 수 있다.
검색은 노트 목록을 필터링하는 방식으로 동작하며, 이번 범위에서는 제목과 본문만 검색 대상으로 사용한다.
최근 검색 기록, 태그 검색, 자동완성은 포함하지 않는다.

## 선택 요약

- 클라이언트 필터링을 사용한다.
  - 이유: 현재 앱은 `NotesContext`에서 전체 노트를 이미 불러오며, 기본 검색은 제목/본문 비교만 필요하다.
- 검색창은 사이드바의 노트 목록 위에 둔다.
  - 이유: 검색 입력과 검색 결과 목록이 같은 영역에 있어 상태 관리와 사용자 흐름이 단순하다.
- 검색어 상태와 동작은 `useNoteSearch` 훅으로 분리한다.
  - 이유: 검색어, debounce, 필터링 결과 계산을 목록 렌더링과 분리해 테스트하기 쉽다.
- 검색 입력 렌더링은 `NoteSearchBox` 컴포넌트로 분리한다.
  - 이유: `NoteList`가 검색 입력 UI 세부 구현보다 검색 결과 목록 렌더링에 집중할 수 있다.
- 검색 대상은 제목과 본문이다.
  - 이유: 현재 PRD 범위는 기본 검색이며, 태그 검색은 별도 기능으로 확장할 수 있게 제외한다.
- 입력 필터링은 200ms debounce 후 반영한다.
  - 이유: 입력 반응성을 유지하면서 매 키 입력마다 필터링하는 일을 줄일 수 있다.

## 대상 화면

- 검색 UI는 `NoteList` 상단에 배치한다.
- 검색 결과는 기존 노트 목록 영역에 표시한다.
- 별도 검색 결과 화면이나 라우트를 만들지 않는다.

## 상태 및 데이터

- 검색어 원본 입력값은 `useNoteSearch` 훅에서 관리한다.
- 필터링에 사용하는 검색어는 200ms debounce가 적용된 값을 사용한다.
- 검색 입력 UI는 `NoteSearchBox` 컴포넌트가 렌더링한다.
- 검색 결과 목록은 `useNoteSearch(notes)`가 반환한 필터링 결과를 사용한다.
- 검색 결과 개수는 필터링 결과 개수를 기준으로 표시한다.
- 검색은 노트 데이터를 변경하지 않는다.

## 주요 동작

- 사용자는 검색창에 검색어를 입력한다.
- 검색어 입력 후 200ms 뒤 노트 목록을 필터링한다.
- 검색어가 비어 있으면 전체 노트 목록을 표시한다.
- 검색어 입력 중에도 현재 선택된 노트와 편집 화면은 유지한다.
- 선택된 노트가 검색 결과에서 사라져도 선택 상태는 유지한다.

## 입력 및 표시 규칙

- 검색어가 없으면 전체 노트를 표시한다.
- 검색어가 있고 결과가 있으면 필터링된 노트 목록을 표시한다.
- 검색어가 있고 결과가 없으면 `검색 결과가 없습니다` 메시지를 표시한다.
- 검색어가 없고 노트도 없으면 기존 `노트가 없습니다` 상태를 유지한다.
- 검색 결과 개수는 기존 노트 개수 표시 영역에 반영한다.
- 검색 입력에는 목적을 알 수 있는 accessible name을 제공한다.

## 검색 규칙

검색은 다음 필드만 대상으로 한다.

- `note.title`
- `note.content`

검색 대상에 포함하지 않는다.

- `id`
- `createdAt`
- `updatedAt`
- `tags`

검색 비교에는 다음 규칙을 적용한다.

- 검색어 앞뒤 공백은 제거한다.
- 대소문자는 구분하지 않는다.
- 제목과 본문 중 하나에 검색어가 포함되면 결과에 포함한다.
- 검색어가 비어 있으면 전체 노트를 결과로 사용한다.

예시:

```text
검색어: " react "
대상: title, content
비교: "react"
```

## 오류 및 예외 처리

- 제목이나 본문이 비어 있으면 빈 문자열로 처리한다.
- 검색어가 있지만 결과가 없으면 오류가 아니라 결과 없음 상태로 처리한다.
- 검색어 변경으로 선택된 노트가 목록에서 사라져도 선택 상태를 초기화하지 않는다.

## 구현 범위

이번 기능에 포함한다.

- `NoteList` 상단 검색창 추가
- `NoteSearchBox` 컴포넌트 분리
- `useNoteSearch` 훅 분리
- 제목과 본문 클라이언트 필터링
- 200ms debounce 적용
- 검색 결과 개수 표시
- 검색 결과 없음 상태 표시
- 검색 중 선택 상태 유지

이번 기능에 포함하지 않는다.

- 서버 검색 API
- 검색 결과 하이라이트
- URL query string 동기화
- 최근 검색 기록 저장
- 최근 검색 기록 드롭다운
- 검색어 자동완성
- 태그 검색
- 태그 전용 필터
- 정렬 옵션 변경
- 검색 결과 별도 화면 또는 별도 라우트
- 검색어별 저장된 뷰
- 본문 전문 검색 인덱스

## 수용 기준

- 사용자는 노트 목록 위 검색창에 검색어를 입력할 수 있다.
- 검색어 입력 후 200ms 뒤 제목과 본문 기준으로 노트 목록이 필터링된다.
- 검색어가 비어 있으면 전체 노트 목록이 표시된다.
- 검색어 앞뒤 공백과 대소문자 차이는 비교에 영향을 주지 않는다.
- 검색 결과 개수는 기존 노트 개수 표시 영역에 반영된다.
- 검색 결과가 없으면 `검색 결과가 없습니다` 메시지가 표시된다.
- 검색어 변경으로 현재 선택된 노트가 결과에서 사라져도 상세/편집 화면은 유지된다.
