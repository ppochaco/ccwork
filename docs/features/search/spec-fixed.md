# 검색 기능 확정 정의서

## 기능 개요

사용자는 검색어로 노트를 찾을 수 있다.
검색은 노트 목록을 필터링하는 방식으로 동작하며, 최근 검색 기록을 확인하고 삭제할 수 있다.

## 선택 요약

- 클라이언트 필터링을 사용한다.
  - 이유: 현재 앱은 `NotesContext`에서 전체 노트를 이미 불러오며, `json-server` 기본 쿼리만으로 제목/본문/태그 대상 검색을 정확히 표현하기 어렵다.
- 검색창은 사이드바의 노트 목록 위에 둔다.
  - 이유: 검색 입력과 검색 결과 목록이 같은 영역에 있어 상태 관리와 사용자 흐름이 단순하다.
- 검색어 상태는 `NoteList` 내부에서 관리한다.
- 검색 대상은 제목, 본문, 태그이다.
  - 이유: 사용자가 노트를 찾을 때 태그도 주요 단서가 될 수 있으며, 앞서 확정한 태그 기능과 자연스럽게 연결된다.
- 검색 기록은 `localStorage`에 저장한다.
  - 이유: 검색 기록은 사용자 브라우저의 편의 기능이며, 노트 데이터 모델과 분리하는 편이 적절하다.
- 입력 필터링은 200ms debounce 후 반영한다.
  - 이유: 입력 반응성을 유지하면서 매 키 입력마다 필터링하는 일을 줄일 수 있다.

## 대상 화면

- 검색 UI는 `NoteList` 상단에 배치한다.
- 검색 결과는 기존 노트 목록 영역에 표시한다.
- 별도 검색 결과 화면이나 라우트를 만들지 않는다.

## 검색 상태

- 검색어 원본 입력값은 `NoteList` 내부 state로 관리한다.
- 필터링에 사용하는 검색어는 200ms debounce가 적용된 값을 사용한다.
- 검색어가 비어 있으면 전체 노트 목록을 표시한다.
- 검색어 입력 중에도 현재 선택된 노트와 편집 화면은 유지한다.

## 검색 대상

검색은 다음 필드만 대상으로 한다.

- `note.title`
- `note.content`
- `note.tags`

검색 대상에 포함하지 않는다.

- `id`
- `createdAt`
- `updatedAt`

## 검색 비교 규칙

- 검색어 앞뒤 공백은 제거한다.
- 대소문자는 구분하지 않는다.
- 태그는 각 태그 문자열에 검색어가 포함되는지 확인한다.
- 검색어가 제목, 본문, 태그 중 하나에 포함되면 결과에 포함한다.

예시:

```text
검색어: " react "
대상: title, content, tags
비교: "react"
```

## 검색 결과 표시

- 검색어가 없으면 전체 노트를 표시한다.
- 검색어가 있고 결과가 있으면 필터링된 노트 목록을 표시한다.
- 검색어가 있고 결과가 없으면 `검색 결과가 없습니다` 메시지를 표시한다.
- 검색 결과 개수는 기존 노트 개수 표시 영역에 반영한다.
- 선택된 노트가 검색 결과에서 사라져도 선택 상태는 유지한다.

## 검색 기록 저장

- 검색 기록은 `localStorage`에 저장한다.
- localStorage key는 `notes.searchHistory`를 사용한다.
- Enter로 검색을 확정했을 때만 검색 기록에 추가한다.
- 빈 검색어는 기록에 추가하지 않는다.
- 검색어는 trim한 값을 기록한다.
- 같은 검색어는 대소문자를 구분하지 않고 중복으로 판단한다.
- 이미 있는 검색어를 다시 검색하면 기존 항목을 제거하고 가장 위로 이동한다.
- 최근 검색어는 최대 10개까지만 보관한다.

## 검색 기록 UI

- 검색 기록은 검색창에 focus가 있을 때 드롭다운으로 표시한다.
- 검색 기록이 없으면 드롭다운을 표시하지 않는다.
- 검색 기록 항목을 클릭하면 해당 검색어를 검색창에 넣고 검색 결과에 반영한다.
- 기록 항목을 클릭하는 동작은 새 기록을 추가하지 않는다.
- 기록 추가는 Enter로 검색을 확정했을 때만 수행한다.
- Escape를 누르거나 검색 영역 바깥을 클릭하면 드롭다운을 닫는다.

## 검색 기록 삭제

- 개별 기록 삭제를 제공한다.
- 전체 기록 삭제를 제공한다.
- 개별 삭제 버튼을 눌렀을 때 검색어 입력값은 변경하지 않는다.
- 전체 삭제를 눌렀을 때 검색어 입력값은 변경하지 않는다.
- 삭제된 기록은 즉시 `localStorage`에 반영한다.

## 키보드 동작

- 검색창에서 Enter를 누르면 현재 검색어를 검색 기록에 추가한다.
- 검색창에서 Escape를 누르면 검색 기록 드롭다운을 닫는다.
- 검색창이 비어 있을 때 Enter를 눌러도 아무 기록도 추가하지 않는다.

## 오류 및 예외 처리

- `localStorage`를 사용할 수 없거나 파싱에 실패하면 검색 기록을 빈 배열로 처리한다.
- 잘못된 검색 기록 데이터는 무시하고 새 기록 저장 시 정상 배열로 덮어쓴다.
- 노트에 `tags` 필드가 없으면 빈 배열로 처리한다.
- 제목이나 본문이 비어 있으면 빈 문자열로 처리한다.

## 구현 범위

이번 기능에 포함한다.

- `NoteList` 상단 검색창 추가
- 제목, 본문, 태그 클라이언트 필터링
- 200ms debounce 적용
- 검색 결과 없음 상태 표시
- localStorage 기반 최근 검색 기록 저장
- 검색 기록 드롭다운 표시
- 검색 기록 개별 삭제
- 검색 기록 전체 삭제
- 기록 클릭 시 검색어 적용
- `tags` 누락 데이터 방어 처리

이번 기능에 포함하지 않는다.

- 서버 검색 API
- 검색 결과 하이라이트
- URL query string 동기화
- 검색 기록 서버 저장
- 검색어 자동완성
- 태그 전용 필터
- 정렬 옵션 변경

## 수용 기준

- 사용자는 노트 목록 위 검색창에 검색어를 입력할 수 있다.
- 검색어 입력 후 200ms 뒤 제목, 본문, 태그 기준으로 노트 목록이 필터링된다.
- 검색어가 비어 있으면 전체 노트 목록이 표시된다.
- 검색 결과가 없으면 `검색 결과가 없습니다` 메시지가 표시된다.
- 검색창에서 Enter를 누르면 검색어가 최근 검색 기록에 저장된다.
- 최근 검색 기록은 최대 10개까지 유지된다.
- 같은 검색어를 다시 저장하면 중복 없이 가장 위로 이동한다.
- 검색창 focus 시 최근 검색 기록 드롭다운을 확인할 수 있다.
- 사용자는 검색 기록을 개별 삭제할 수 있다.
- 사용자는 검색 기록을 전체 삭제할 수 있다.
- 검색어 변경으로 현재 선택된 노트가 결과에서 사라져도 상세/편집 화면은 유지된다.
